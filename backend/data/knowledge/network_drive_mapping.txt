Title: Network Drive Mapping Guide

Overview
Corporate network drives are hosted on the on-premises file server (\\fileserver.company.internal) and on SharePoint Online. Drives are mapped automatically via Group Policy for domain-joined Windows devices. macOS users must map drives manually.

Standard Drive Mappings (Windows — Auto-applied via Group Policy)
- H: — Home drive (personal, 10 GB quota, backed up nightly)
- S: — Shared departmental drive (read/write for your department, read-only for others)
- P: — Project drives (access by project membership — request via IT portal)
- R: — Read-only reference materials (company-wide read access)

Manually Mapping a Drive on Windows
If automatic mapping fails:
1. Open File Explorer and click "This PC".
2. Click "Map network drive" in the toolbar.
3. Choose a drive letter and enter the path, e.g., \\fileserver.company.internal\shared.
4. Tick "Reconnect at sign-in" and "Connect using different credentials" if prompted.
5. Enter your corporate credentials (domain\username format) if asked.
6. Click Finish. The drive will appear under "This PC".

Mapping a Drive on macOS
1. In Finder, press Cmd+K to open "Connect to Server".
2. Enter the server address: smb://fileserver.company.internal/shared
3. Click Connect. Enter your corporate username and password when prompted.
4. The share will mount under /Volumes/.
5. To persist after reboot, open System Settings > General > Login Items and add the mounted share.

Accessing SharePoint Document Libraries as Drives
SharePoint libraries can be synced locally using the OneDrive sync client:
1. Navigate to the SharePoint site in a browser.
2. Click "Documents" in the left sidebar.
3. Click "Sync" in the toolbar.
4. Confirm in the OneDrive dialog. Files will sync to your local OneDrive folder.

Troubleshooting
- "Network path not found": Ensure you are connected to the corporate network or VPN.
- "Access denied": Your account may lack permissions to the share. Submit an access request via the IT portal specifying the share path and business reason.
- Slow access: Large files should be accessed over wired ethernet, not Wi-Fi. Use the VPN's split-tunnel exempt list for SMB shares (contact Network team).
- Drive disconnects on sleep: Enable "Keep network drives connected" in Power Options (Windows) or increase the SMB keepalive setting on macOS.

Quotas and Storage Requests
Home drive quota increases require manager approval. Submit requests via the IT portal with a business justification. Departmental shared drive quotas are managed by department heads — contact your department's IT Liaison.
